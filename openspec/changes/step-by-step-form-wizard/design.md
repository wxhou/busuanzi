## Context

当前 BaziForm 组件是一个单页表单，包含9个字段（姓名、性别、出生年份、年柱、月柱、日柱、时柱、起运年龄、第一大运）。用户需要一次性填写所有信息，对新用户不够友好。

## Goals / Non-Goals

**Goals:**
- 将表单拆分为两个步骤，降低用户输入负担
- 每步骤完成后进行字段校验
- 保持现有表单的样式和功能不变

**Non-Goals:**
- 不添加示例数据功能
- 不修改后端服务或 API
- 不添加动画过渡效果（后续可扩展）

## Decisions

### 1. 表单状态管理
- **方案**: 使用 BaziForm 内部的 `useState` 管理当前步骤
- **理由**: 父组件只需要最终数据，不需要感知中间步骤

```typescript
// 内部状态
const [currentStep, setCurrentStep] = useState(1);
const [formData, setFormData] = useState<UserInput>({...});
```

### 2. 步骤划分
- **步骤1**: 姓名(name)、性别(gender)
- **步骤2**: 出生年份、年柱、月柱、日柱、时柱、起运年龄、第一大运

### 3. 步骤指示器
- **方案**: 简单的步骤指示器组件，显示1/2、2/2进度
- **设计**:
  ```
  ●─────○
  步骤1  步骤2
  ```

### 4. 导航按钮
- **步骤1**: 显示"下一步"按钮
- **步骤2**: 显示"上一步"和"生成人生K线"按钮

### 5. 字段校验
- **方案**: 点击下一步时校验当前步骤的必填字段
- **实现**: 检查对应字段是否为非空字符串

## Risks / Trade-offs

| 风险 |  Mitigation |
|------|-------------|
| 步骤2校验失败后返回步骤1，再进入步骤2数据丢失 | 使用单一 formData 状态管理，步骤切换不重置数据 |
| 移动端步骤指示器占用空间 | 简化移动端指示器，只显示当前步骤编号 |
