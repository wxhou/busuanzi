# Life-K-Line 项目开发提示词

## 项目概述
Life-K-Line 是一个基于八字命理的可视化分析工具，通过调用外部 AI 服务生成人生 K 线图和命理分析报告。

## 技术栈说明

### 前端框架
- **React 19.0.0**: 核心前端框架，用于构建用户界面
- **TypeScript 5.2**: 提供静态类型检查，增强代码质量和可维护性
- **Vite 5.2.0**: 现代化构建工具，提供快速的开发服务器和优化的打包功能

### UI 和样式
- **Tailwind CSS**: 实用优先的 CSS 框架，用于快速构建自定义用户界面
- **Lucide React**: 提供精美的 SVG 图标集
- **Recharts**: 基于 D3.js 的声明式图表库，用于创建人生 K 线图

### 架构特点
- **无服务器架构**: 项目不包含传统后端，通过 API 调用外部服务
- **AI 集成**: 通过 OpenRouter API 调用 Google Gemini 模型进行八字命理分析
- **组件化开发**: 使用 React 组件化架构，提高代码复用性和可维护性

### 核心文件结构
```
.
├── components/                 # React 组件
│   ├── BaziForm.tsx           # 八字输入表单
│   ├── LifeKLineChart.tsx     # K 线图表组件
│   └── AnalysisResult.tsx     # 分析结果展示
├── services/                  # 服务层
│   └── geminiService.ts       # AI 服务接口
├── App.tsx                    # 应用根组件
├── index.tsx                  # 应用入口文件
├── constants.ts               # 常量配置
├── types.ts                   # TypeScript 类型定义
└── vite.config.ts             # Vite 配置文件
```

### 开发命令
- `npm run dev`: 启动开发服务器
- `npm run build`: 构建生产版本
- `npm run preview`: 预览生产构建

### 关键实现要点

#### 1. AI 服务集成
- 使用 OpenRouter API 调用 Google Gemini 模型
- 通过系统指令和用户提示词生成结构化的八字分析结果
- 实现错误处理和数据验证机制

#### 2. 数据结构设计
- 定义完整的 TypeScript 接口用于类型安全
- 设计支持 K 线图展示的数据结构
- 包含评分机制的多维度分析结果

#### 3. 用户体验优化
- 响应式设计适配不同屏幕尺寸
- 加载状态和错误处理
- 清晰的用户交互反馈

### 扩展开发指南

#### 添加新功能
1. 遵循组件化原则，创建独立的可复用组件
2. 使用 TypeScript 定义清晰的接口和类型
3. 集成到现有状态管理流程中
4. 保持 UI 风格的一致性

#### 性能优化
- 使用 React.memo 进行组件记忆化
- 实现虚拟滚动处理大量数据
- 优化 API 调用频率和缓存策略
- 代码分割和懒加载

#### 代码规范
- 使用函数式组件和 Hooks
- 遵循 React 最佳实践
- 保持一致的代码风格和命名约定
- 充分利用 TypeScript 的类型检查能力

### 部署说明
- 使用 Vite 构建静态资源
- 可部署到任何静态文件托管服务
- 确保环境变量中包含正确的 API 密钥配置
